# Task 1.1: Add Semicolon Statement Separator

## Objective

Allow `;` as a statement separator in flowchart diagrams, so `A --> B; B --> C` works on a single line. This is standard Mermaid syntax that many real-world diagrams use.

## Location

Modify: `src/parser/grammar.pest`
Modify: `tests/integration.rs` (or parser unit tests)

## TDD Phases

### Red: Write Failing Tests

```rust
#[test]
fn test_semicolon_separator_two_statements() {
    let input = "graph TD\nA --> B; B --> C\n";
    let result = parse_flowchart(input).unwrap();
    assert_eq!(result.edges().len(), 2);
}

#[test]
fn test_semicolon_separator_mixed_with_newlines() {
    let input = "graph TD\nA --> B;\nB --> C\n";
    let result = parse_flowchart(input).unwrap();
    assert_eq!(result.edges().len(), 2);
}

#[test]
fn test_semicolon_separator_multiple() {
    let input = "graph TD\nA --> B; B --> C; C --> D\n";
    let result = parse_flowchart(input).unwrap();
    assert_eq!(result.edges().len(), 3);
}
```

**What the failing test asserts:** Multiple statements separated by `;` on a single line parse into separate edges.
**Expected failure reason:** The grammar currently only recognizes `NEWLINE` as a statement separator, so `;`-separated statements fail to parse.

### Green: Minimal Implementation

Update the `flowchart` rule in `grammar.pest` to accept `;` alongside `NEWLINE` as a statement separator:

```pest
separator = _{ NEWLINE_CHAR | ";" }
```

Update the statement-list rule to use `separator` instead of `NEWLINE_CHAR` wherever statement boundaries are recognized.

### Refactor: Clean Up

- Ensure trailing semicolons and redundant separators (`;\n`, `;;`) don't cause errors
- Verify all existing tests still pass

## Context

The upstream Jison flowchart grammar uses `;` as a valid statement terminator. This is one of the most common compatibility gaps when users paste real Mermaid diagrams into mmdflux.
