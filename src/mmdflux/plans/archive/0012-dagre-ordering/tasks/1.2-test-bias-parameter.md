# Task 1.2: Add Unit Test for Bias Parameter

## Objective
Verify that `bias_right = true` produces a different ordering than `bias_right = false` when nodes have equal barycenters.

## Location
Modify: `src/dagre/order.rs` (test module)

## Implementation

```rust
#[test]
fn test_bias_right_changes_order() {
    // Create a graph where two nodes in layer 1 have the same single
    // predecessor in layer 0, producing equal barycenters.
    //
    //   A
    //  / \
    // B   C
    //
    // Both B and C have barycenter = order(A) = 0.
    // With left bias: B(original_pos=0) before C(original_pos=1)
    // With right bias: C(original_pos=1) before B(original_pos=0)

    let mut graph: DiGraph<()> = DiGraph::new();
    graph.add_node("A", ());
    graph.add_node("B", ());
    graph.add_node("C", ());
    graph.add_edge("A", "B");
    graph.add_edge("A", "C");

    let mut lg = LayoutGraph::from_digraph(&graph, |_, _| (10.0, 10.0));
    rank::run(&mut lg);
    rank::normalize(&mut lg);

    // Initialize orders
    let layers = rank::by_rank(&lg);
    for layer in &layers {
        for (idx, &node) in layer.iter().enumerate() {
            lg.order[node] = idx;
        }
    }

    let edges = lg.effective_edges();
    let fixed = &layers[0]; // [A]
    let free = &layers[1];  // [B, C]

    // Left bias (bias_right = false)
    reorder_layer(&mut lg, fixed, free, &edges, true, false);
    let left_order_b = lg.order[1]; // B's index
    let left_order_c = lg.order[2]; // C's index

    // Reset orders
    for (idx, &node) in free.iter().enumerate() {
        lg.order[node] = idx;
    }

    // Right bias (bias_right = true)
    reorder_layer(&mut lg, fixed, free, &edges, true, true);
    let right_order_b = lg.order[1];
    let right_order_c = lg.order[2];

    // Left bias: B before C (smaller original_pos wins)
    assert!(left_order_b < left_order_c, "Left bias should put B before C");
    // Right bias: C before B (larger original_pos wins)
    assert!(right_order_b > right_order_c, "Right bias should put C before B");
}
```

## Context

Node indices depend on `LayoutGraph::from_digraph` insertion order. The test may need to look up node indices by name rather than hardcoding `1` and `2`. Check how `setup_graph_and_run` in the existing tests handles this -- it uses `DiGraph::add_node` in order, so indices should be sequential.

The `reorder_layer` function is private (`fn`, not `pub fn`), so this test must be within the `mod tests` block inside `order.rs`.
