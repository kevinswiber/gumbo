# Task 1.3: Add `spread_points_on_face()` Utility

## Objective

Add a function that computes N evenly-spaced attachment positions along a node face.

## Location

Modify: `src/render/intersect.rs`

## Implementation

```rust
/// Compute N evenly-spaced attachment positions along a face.
///
/// Uses N+1 divisions to avoid placing edges at face corners.
/// For N=1, returns the center of the extent.
/// Returns (x, y) coordinates.
pub fn spread_points_on_face(
    face: NodeFace,
    fixed_coord: usize,
    extent: (usize, usize),
    count: usize,
) -> Vec<(usize, usize)> {
    if count == 0 {
        return vec![];
    }

    let (start, end) = extent;
    let range = end.saturating_sub(start);

    if count == 1 {
        let mid = start + range / 2;
        return match face {
            NodeFace::Top | NodeFace::Bottom => vec![(mid, fixed_coord)],
            NodeFace::Left | NodeFace::Right => vec![(fixed_coord, mid)],
        };
    }

    (0..count)
        .map(|i| {
            let pos = start + ((i + 1) * range) / (count + 1);
            let pos = pos.min(end);
            match face {
                NodeFace::Top | NodeFace::Bottom => (pos, fixed_coord),
                NodeFace::Left | NodeFace::Right => (fixed_coord, pos),
            }
        })
        .collect()
}
```

## Context

- The N+1 division formula `start + (i+1) * range / (N+1)` places edges at fractional positions that avoid the face corners:
  - N=2 on range 0..8: positions at 8/3≈2 and 2*8/3≈5
  - N=3 on range 0..8: positions at 2, 4, 6
- For N=1, returns the center (unchanged from current behavior, where `intersect_rect()` produces a center attachment for a single edge).
- The `pos.min(end)` clamp handles integer rounding near the upper bound.
- For very narrow nodes (e.g., 5-wide with 3 usable columns and 3 edges), the formula gives columns 0, 1, 2 relative to start — packed tightly but each getting a distinct cell. This is the best possible outcome given the constraint.

## Acceptance Criteria

- [ ] N=0 returns empty vec
- [ ] N=1 returns center of extent
- [ ] N=2 on range (0, 8) returns points at approximately (2, fixed) and (5, fixed) for Top/Bottom
- [ ] N=3 on range (0, 8) returns approximately (2, fixed), (4, fixed), (6, fixed)
- [ ] All returned points are within [start, end]
- [ ] Left/Right faces swap coordinate axes correctly (returns (fixed, pos) instead of (pos, fixed))
- [ ] Unit tests covering N=1,2,3,5 and narrow range (range < count)
